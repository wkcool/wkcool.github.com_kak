连 Tom Warren 都放弃的 Windows Phone，官方应用荒还在发酵

日期  2014年12月22日 11:00 由  36kr 编辑 新闻类别  互联网 标签 column
  

摘要 : 连 Tom Warren 都放弃的 Windows Phone，官方应用荒还在发酵人称 iVerge 的知名科技媒体里一直有一位和网站整体偏好格格不入的微软观察家 —— Tom Warren，他在入职 The Verge 之前是 WinRumors 的创始人。Windows Phone 主流应用不完全、新兴应用跟进缓慢的情况存在已久，而现在 Snapchat 的不巧的是，Tom Warren 上周在 The Verge 主站和自己的 Twitter 账户上公开表示自己已经再也受不了 Windows Phone 应用短缺的状况，打算转而使用 iPhone。其中受波及最严重的是 6snap，一个出自知名第三方应用开发者 Rudy Huyn 的应用。Rudy Huyn 最早以第三方 Vine 客户端 6sec 和第三方 Instagram 客户端 6tag （彼时叫 6tagram）闻名，他热衷于将其它平台热门的应用以个人风格移植到 Windows Phone 平台，同时保持着一流的性能和设计水平。Windows Phone 官方应用荒的尴尬局面已经持续很久了。虽然微软方面一直表现出会派驻官方技术支持人员协助主流公司移植应用的态度，但实际状况并不理想。同样由微软团队深度介入的还有官方 Instagram 客户端，一个延期近两年上架并且至今未摆脱 "beta" 标签的官方应用。，与它情况类似的还有 Flipboard。Windows Phone 上公认最优秀的第三方 Twitter 客户端 Tweetium 出自前微软员工之手，公认最优秀的第三方 Reddit 客户端之一 Baconit 出自微软在职员工。微软的人自身成了这个平台的天花板，而他们所能做到的相对其它平台而言只算“仅此而已”。这些国内 Windows Phone 用户和早期魅族的粉丝一样，会主动找到其它平台的用户、论坛，主动挑起争执并彼此抱团。造成今天这个局面的是个一步步由微软自己建立起来的循环。而所谓“低端市场”的开发者回报率和用户容忍度你们也看到了。

人称 iVerge 的知名科技媒体里一直有一位和网站整体偏好格格不入的微软观察家 —— Tom Warren，他在入职 The Verge 之前是 WinRumors 的创始人。不巧的是，Tom Warren 上周在 The Verge 主站和自己的 Twitter 账户上公开表示自己已经再也受不了 Windows Phone 应用短缺的状况，打算转而使用 iPhone。Windows Phone 主流应用不完全、新兴应用跟进缓慢的情况存在已久，而现在 Snapchat 的

会让这一情况更加尴尬。

就在昨天，Windows Phone Store 中所有第三方 Snapchat 应用都遭到下架，这件事当然和 Snapchat 官方脱不了关系。其中受波及最严重的是 6snap，一个出自知名第三方应用开发者 Rudy Huyn 的应用。Rudy Huyn 最早以第三方 Vine 客户端 6sec 和第三方 Instagram 客户端 6tag （彼时叫 6tagram）闻名，他热衷于将其它平台热门的应用以个人风格移植到 Windows Phone 平台，同时保持着一流的性能和设计水平。微软官方从未正面对 Rudy Huyn 的移植行为表过态，但他的第三方应用在 Windows Phone Store 的审核流程一直很顺利，Windows Phone 8.1 的新自定义锁屏 API 也由他与微软官方合作完成首个 demo。

这也并不是 Rudy Huyn 第一次与原软件开发商出现争执：6snap 之前是他的第三方 Tinder 客户端 6tinder，这款应用由于 Rudy Huyn 与原开发商迟迟无法达成一致而反复多次上下架。此前 Telegram 和国内的百度贴吧都选择将某个优秀的第三方客户端收作官方客户端维护，但 Tinder 和 Snapchat 显然不愿意这么做。

Windows Phone 官方应用荒的尴尬局面已经持续很久了。虽然微软方面一直表现出会派驻官方技术支持人员协助主流公司移植应用的态度，但实际状况并不理想。以国内的微博为例，微博 Windows Phone 客户端在发展初期就有微软常驻人员提供支持，但无论是在设计、功能还是稳定性方面都甚至不如 Windows Phone 平台的主流应用（这一状况在近期因为重组团队有所好转）。同样由微软团队深度介入的还有官方 Instagram 客户端，一个延期近两年上架并且至今未摆脱 "beta" 标签的官方应用。，与它情况类似的还有 Flipboard。

事实上，微软对 Windows Phone 应用表现出来的态度一直让人难以捉摸。上文提到的 Tom Warren 曾经在 Twitter 中放出过一组对比图片，用以说明 Windows Phone 平台的 Office 在功能和使用体验上都远不如 iOS 版本。一系列的情况都不免让人怀疑，出于某种原因，连微软自己都无法在现有的 Windows Phone 平台上完成足够优秀的应用。Windows Phone 上公认最优秀的第三方 Twitter 客户端 Tweetium 出自前微软员工之手，公认最优秀的第三方 Reddit 客户端之一 Baconit 出自微软在职员工。微软的人自身成了这个平台的天花板，而他们所能做到的相对其它平台而言只算“仅此而已”。微软官方甚至还先后两次分别鼓励开发者：1.将网页（Web App）打包成应用上传至 Windows Phone Store 2.上传尽可能多的应用以换取不同等级奖励。两次活动都直接造成商店中低质、山寨应用泛滥的局面。BI 曾经发表过一份报告，Windows Phone Store 中光低质、虚假 Facebook 客户端就占很大一部分。微软虽然曾经更新商城政策，禁止以热门应用名称加以 "guide"、"handbook"、"features" 等字样发布虚假应用，但直至今日仍然可以在商城官方的最新应用推荐里看到这类应用。

当然也有个别积极的案例：比如黑莓积极地在 Windows Phone 平台上推出 BBM 和 BES 应用，自身发展缓慢的 Path 在 Windows Phone 平台上甚至推出重新设计的官方应用，在其它平台上备受争议的 QQ 针对 Windows Phone 和 Windows 平台融入自己的交互设计语言并且程序整体水平惊人的高，另外平台上知名的第三方应用 Poki 还受到过 Pocket 官方 Twitter 账号的推荐。至于 Telegram 和 百度贴吧的曲线救国态度，我们也已经提到过。

在微软之外，还有很大一部分问题出在用户身上。有不止一个科技领域相关从业者跟我说过类似“有些国内 Windows Phone 用户总让人觉得戾气很重”的话。这些国内 Windows Phone 用户和早期魅族的粉丝一样，会主动找到其它平台的用户、论坛，主动挑起争执并彼此抱团。我也曾切身听过/见证过不止一个这样案例：开发商在社区或其它反馈渠道的声音推动下终于决定开发 Windows Phone 版本应用，但他们对 Windows Phone 平台的摸索以及产品必须的迭代周期并不被用户包容，最终项目在应用商城和微博等社交网络的如潮恶评之下不得不终止。因为同样的情况，poki 开发者也在 Twitter 上表达过要从中国区商城下架应用的意愿。

造成今天这个局面的是个一步步由微软自己建立起来的循环。早期 Windows Phone 7 明显不成熟的情况下选择发布这款系统和多款旗舰手机，这个 Windows Phone 7 生态苦苦挣扎了近两年，将消费者和开发者的信心及新鲜感消磨殆尽，以至于 Windows Phone 8 开始微软一直宣称会把重心放在低端市场。而所谓“低端市场”的开发者回报率和用户容忍度你们也看到了。也许，微软当初没有把（由今天名满海内外的 Rigo Design 和 Ark Design 创始人参与的）Windows Mobile 7 仓促换成 Windows Phone 7 发布，再让相对成熟的 Windows Phone 8 直接面市，情况就会因此不同。又或者，如 Tom Warren 所说，希望 Windows 10 会是真正的转变契机。
