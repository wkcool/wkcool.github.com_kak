分布式文档型

SequoiaDB

透露已经完成了千万美元级A轮融资，领投机构为启明创投，这也是国内首家被顶级美元基金投资的新型数据库软件公司。

Oracle

SQL Server

是三家牢牢占据前排的传统数据库巨头，这些传统数据库都是关系型数据库，但2009起开始逐渐萌发的NoSQL运动让人们看到了另一种数据库架构的可能性。

传统关系型数据库发展时间久，性能稳定安全，这些都是优点。但是面对web 2.0时代的大数据读写、高并发访问以及频繁的横向扩展需求，关系型数据库因为表结构不易更改、扩展性差等原因稍显疲软。NoSQL则舍弃数据间的关系特性，以更灵活的数据组织形式和分布式存储来达到满足扩展需求的目的。

NoSQL目前还没有一个很完善的定义（No-SQL或者Not-Only-SQL都不够准确，non-relational相对好点），但在国外已经有一批发展不错的NoSQL公司了，比如

MongoDB

、couchDB、Redis、MarkLogic等。新浪微博、淘宝等平台也已经在相继使用NoSQL作为基础架构，但国内专门开发NoSQL基础数据库软件的似乎很少，除了BAT可能自己有在研发，SequoiaDB似乎是少数一家专门做NoSQL的。

SequoiaDB是一个国产的分布式数据库，和北美的MongoDB一样，属于文档类NoSQL数据库。与MongoDB不同的是，SequoiaDB针对企业级用户，加入了其他NoSQL公司少有的事务操作功能，同时封装了用来支持

Hadoop

的MapReduce调用接口，此外，SequoiaDB还提供连接SQL的插件，让习惯用SQL某些功能的企业用户可以在SequoiaDB里对接SQL。

“国外这些相同的产品都非常新，基本最老的也就是5年上下，所以和这些产品相比，我们的起步基本是在同一阶段，可能仅比人家稍微晚个2年左右。”SequoiaDB的联合创始人王涛先生向

介绍道，“不过我们有后发优势，在研发过程中我们一直关注业界对这些友商产品的评论和需求，发现一些功能和性能上的不足，我们在设计产品的时候就能够迅速弥补上。”

SequoiaDB的事务处理功能就是这样来的：在和客户民生银行的洽谈中，王涛发现企业在架设NoSQL数据库时对事务处理的需求还是很大的，由于NoSQL为了扩展牺牲了acid一致性，加上CPA定理，许多NoSQL产品是不支持事务处理的。王涛认为这对于企业级用户会是一个非常大的痛点，SequoiaDB对事务的支持也就顺理成章了。

王涛创业前曾是IBM公司DB2北美实验室全球最高顾问小组成员之一（全球仅15人），在IBM里由于看到了NoSQL可能对数据库产生颠覆性创新，便决定出来创业。我请他谈谈对NoSQL这个行业的把握和看法，他表示：

“NoSQL和关系型数据库会有一个长期的共存，分别适用于不同的应用领域。现在很多传统的企业也都开始慢慢接受互联网的思想，包括其业务模式以及后台所采用的技术，包括NoSQL数据库。我认为不管是NoSQL数据库还是传统关系型数据库都是会继续发展变化的，而未来分布式架构一定是数据库变革的一个方向，现在的NoSQL就是这样一个雏形。慢慢NoSQL的功能随着时间推移会越来越强大，早晚与传统数据库会有一个融合的趋势。而这种融合，我认为会是NoSQL逐渐包容关系型数据库，而不是关系型数据库反过来包容NoSQL。毕竟关系型数据库的设计和研发理念都已经非常陈旧了，而NoSQL则是一个分布式数据库的趋势方向。”

，SequoiaDB目前的企业级用户，在传统企业里有包括民生银行、海南航空、电信移动等；而互联网行业里面也有像蓝汛、蓝港在线这类企业。民生银行已经有多个系统在线运行，包括存储票据影像和日志系统等。

未来SequoiaDB打算在丰富自己的功能的同时，更多的与生态圈里的其他友商产生联系，因为数据库这类基础架构太过庞大，自己单独玩是肯定行不通的。

在采访的最后，王涛告诉

，SequoiaDB将在最近开源。他认为开源是整个行业的趋势，软件卖授权对新型数据库已经不适用了，开源+原厂支持的企业级服务才是根本。这家坐落在广州的公司目前拥有一个30多人的团队，研发工程师队伍主要来自IBM和华为。
